Workload Distribution (2026)

A real‑time dashboard that extracts and visualizes team workload directly from a Jira project view. It consolidates paginated ticket data into a single interactive panel, providing instant visibility into how work is distributed across the team and who has available capacity.

Core capabilities:

  • Unified data view: automatically aggregates tickets from across paginated tables, presenting every team member and their current assignments in one place, eliminating the need to click through multiple pages or manually track counts
  • Team capacity indicators: color‑coded stats immediately reveal who is busy and who has workload availability, turning raw ticket data into actionable insight
  • One‑click person filtering: click any team member's name to instantly isolate their tickets in the main table, ideal for quick progress checks without manual searching
  • Status‑based navigation: toggle between workflow stages via dedicated buttons, each showing the total ticket count for that status to maintain context while switching views
  • Dynamic workload metrics: switch between raw ticket counts and percentage‑of‑status distribution to understand both absolute volume and relative team share
  • Focused team scope: displays data exclusively for team members, excluding unrelated assignees to ensure the dashboard always reflects your actual team composition
  • Interactive data table: sort any column with a single click; headers stay visible while scrolling

✓ Zero infrastructure: no installation, no backend dependencies, no system admin approvals

JIRA SyncFlow (2026)

This tool centralizes an existing, multi-step ticketing workflow into a single interface. It accelerates the JIRA review and pre-approval stages by reducing context switching, guiding key decision points, and supporting accurate, consistent communications. It requires no backend services or environment setup.

Operational enhancements:

  • Consolidated workspace: eliminates the need to search across multiple sections of a ticket and navigate multiple platforms (Jira, Outlook, and AEM)
  • Workflow friction reduction: removes unnecessary steps by integrating multiple tasks into a single interface, helping move tickets through review and approval stages faster
  • Adaptive workflow guidance: detects the current ticket status and suggests the appropriate transition action, including safe rollback options when reversions are required
  • Status timer indicator: shows how long a ticket has remained in its current stage, providing immediate insight into issues that are stalled, delayed, or overdue
  • One-click page access: opens all listed staging URLs instantly, either in the AEM editor or preview environment, without copying links individually or switching platforms
  • Email draft generator: pre-fills recipients, subject lines, and relevant URLs, letting users switch between preview links for client approval or live links for final distribution

✓ Zero infrastructure: no installation, no backend dependencies, no system admin approvals

DocPrep (2026)

An application that provides pre-production analysis of Microsoft Word documents, identifying accessibility and structural issues while delivering actionable diagnostics through automated validation and accessibility scoring. Key Features:

  • Comprehensive accessibility audit: detects missing alt text, malformed headings and lists, table structure errors, empty and colored background cells, and formatting issues (all caps, underline, italic usage)
  • Smart link verification: validates external links in real-time, flags local/email links, and identifies same-text-different-URL inconsistencies
  • Document health check: flags tracked changes and comments that impact diagnostic accuracy
  • Priority-driven workflow: categorizes fixes by impact (high, medium, low) with detailed counts for each category
  • Accessibility score: calculates an overall accessibility percentage and provides detailed health metrics for each content category
  • Interactive diagnostic reports: generates a Word file featuring a TOC with CTRL+clickable links to flagged issues for review and correction
  • Privacy-first analysis: the entire document is processed locally for security; only URLs are sent externally for link validation
  • Tech stack: JavaScript, Microsoft Azure Functions (Python), HTML, CSS

XML Morph (2023)

This tool effortlessly transforms well-structured HTML into custom XML format used in TBS policy documents, streamlining the conversion process. Here's a summary of its main functions and behavior:

  • Initial storage: stores the user's original pasted content so it can be reverted or compared later
  • HTML to XML conversion: converts HTML into a custom XML format that includes structural elements like chapters, appendices, clauses, and sections. it also adds headers, handles special characters, and replaces standard list tags with more semantic equivalents
  • XML to HTML conversion: reverses the above process, stripping out XML wrappers and reconstructing readable HTML.
  • List handling: distinguishes between clause-style lists and normal lists to ensure only the appropriate ones are converted to <clause> elements
  • Section wrapping: automatically groups headings and their following content into semantic sections based on heading levels
  • Preview and error display: renders color-coded structured output and displays error messages or a parsed view as needed
  • Cleanup: removes style elements and ensures self-closing tags are handled correctly
  • Validation: checks for matching and balanced HTML/XML tags, reporting issues if any tags are unbalanced or improperly structured

CodeBOT (2020)

Analytics-driven automated HTML editor that opens Word documents of all sizes while applying the Canadian Standard on Web Accessibility. CodeBOT combines a sequential processing pipeline from core syntax cleanup to content structuring, streamlining the conversion workflow and supporting WCAG compliance through the following key operations:

  • Syntax cleanup: removes proprietary Microsoft markup, deprecated attributes, extra spaces, line breaks, and empty tags for cleaner, standards-compliant HTML
  • Dynamic language detection: applies language-specific formatting to English or French content
  • Content structuring: builds table of contents, corrects malformed structural elements, encodes images and bilingual footnotes, adds semantic section tags, and formats definition lists
  • Table accessibility: allows users to navigate each table either directly within the editor or through a modal preview built on a Shadow DOM architecture. It applies proper table markup (<thead>, <tbody>, <tfoot>), sets auto-scope attributes, and enables toggling of basic formatting options
  • URL processing: decodes Outlook Safe Links, converts absolute URLs to relative paths, and provides a Link Status Report that identifies broken links
  • Analytics and feedback: provides real-time counts displayed beside each operation, allowing users to immediately see how many instances of specific issues or elements (such as deprecated attributes or footnotes) are present in the HTML. This helps users quickly assess the scope of cleanup and improvements needed
  • Real-time validation: automatically detects HTML syntax errors, invalid heading hierarchy structure, accessibility errors, and displays them in the console panel, with each message clickable to jump directly to the specific line number
  • Tech stack: JavaScript, HTML, CSS, CodeMirror 5.65.2, Google Apps Script (serverless backend)
  • System Architecture: discover the visual overview illustrating the sequential pipeline and automated processing flow

Modules added since rollout

  • TableNote Pro (2022): an enhanced version of TableNote (2021), a tool for auto-generating footnotes in HTML tables. It provides new options such as toggling footnotes on or off, visual highlighting to identify and trace footnote references, and the ability to inject the footnote section directly within the table
  • Link Validator (2024): a utility that runs in parallel yet independently of main editing tasks. It scans all hyperlinks to detect broken, redirected, or malformed URLs and generates a detailed status report complete with visual indicators, copy/export tools, and an HTTP status legend. It features a 3-slot FIFO popup architecture ensuring only the latest reports are visible. Built from scratch, this component can be integrated into other systems with minimal modification
  • Paste from external sources (2025): this feature expands CodeBOT’s capabilities beyond its original support for .docx files and plain text, allowing users to copy and paste content from a variety of sources (Word, Excel, web pages, Google Sheets, etc.) directly into the editor while preserving structure and formatting
  • Link Validator (2025 – Redesigned): a fully revamped system featuring a dedicated Link Validation section, Google Sheets export, Outlook email integration, and archive management, streamlining workflow and enhancing user focus

AutoPDF Builder (2020)

This application automates a previously manual time-consuming task by converting web pages into bilingual PDF files. It reads URLs from a spreadsheet and processes each one through the following steps:

  • Loads the web page and prepares it for printing by expanding all collapsible elements (<details> tags), ensuring that all content is visible
  • Removes non-essential elements like breadcrumb navigation and archived notices to produce a cleaner and more focused print output
  • Automatically configures the browser's print settings (e.g., layout, margins, headers/footers) to ensure consistent PDF formatting
  • Saves the English version of the page as a PDF, then switches to the corresponding French page to repeat the preparation and PDF conversion steps, producing a bilingual counterpart
  • Tech stack: Python, Selenium WebDriver, Chrome browser, JavaScript, Microsoft Excel

Flash-Conv (2019)

Batch-driven desktop application that converts Word documents both new and old legacy formats (doc and docx) to clean, valid HTML5, automating a typically tedious manual process common in government departments. Key features:

  • Batch processing: supports single-file or batch conversion from either a ZIP archive or directory, handling of large volumes with minimal user input
  • Analytics: uses data science concepts to analyze HTML output by tabulating content, visualizing patterns with graphs, and applying mathematical methods to assess the complexity of the conversion process. This analytical insight helps identify challenges, optimize workflows, and improve overall conversion quality and efficiency
  • Performance metrics: efficiently processes batches of up to 1,000 files in about 10 minutes with analytics feature enabled, or 2.5 minutes without analytics
  • Corrupted file recovery: automatically flags and converts damaged files that cannot be opened manually
  • Outlook integration: results are emailed automatically via Outlook for streamlined delivery
  • Tech stack: Python, JavaScript, Vue.js, Chart.js, Win32com.client, HTML, CSS, Headless Chrome

TableXtract (2025)

Cloud-based microservice that converts table images into structured HTML. It accepts binary image uploads, performs server-side OCR and table detection, and returns clean, standardized output ready for downstream automation. The service runs in both local and web-based environments, delivering fast and consistent results. Built as a scalable, serverless component, TableXtract can be easily integrated into existing applications and workflows that require accurate table extraction from screenshots, scans, or other image sources. Core features:

  • Smart table recognition: automatically identifies tables and their structure in images using AI-powered analysis
  • Spell check integration: flags potential spelling errors in extracted text with visual highlighting and suggested corrections
  • Clean HTML output: produces well-structured, accessible HTML tables ready for direct use or automation
  • Flexible output options: provides easy ways to copy, download, and preview results, including a code view toggle for developer workflows
  • Drag-and-drop interface: supports intuitive file uploads with instant preview for common image formats (JPEG, PNG, BMP, GIF)
  • Tech stack: Microsoft Azure (serverless backend), Python, HTML, JavaScript, CSS

TableCraft (2024)

Originally designed with financial data in mind, this tool provides a graphical interface for editing and customizing HTML tables. It enables quick styling and structural changes, while also managing accessibility attributes without the need to write HTML code. Here's a breakdown of what the application does:

  • Real-time preview: displays a live preview of your HTML table as you type, showing changes instantly
  • Interactive editing: users can click directly on table cells, columns, or the caption in the preview to apply changes
  • Flexible operations: ability to choose from various modes to edit single cells, entire columns, or the caption, and apply styles, structure changes, or accessibility attributes
  • Undo and cleanup: easily reverse recent modifications or clean up the HTML by removing unnecessary tags or classes
  • Code management: copy the cleaned HTML to your clipboard or reset the table to its initial state

TableSync Mirror (2024)

This tool easily synchronizes two HTML tables by transferring only the updated content from one table to another, while preserving the original formatting and structure.

TableNote (2021)

This tool simplifies the process of adding footnotes to your HTML tables by automatically generating and formatting them. It offers an efficient solution for numbering footnotes, applying symbols, and managing multi-referenced footnotes. Features overview:

  • Generates footnotes wrapped in <sup> tags
  • Processes numbers sequentially followed by symbols
  • Supports multi-referenced footnotes
  • Automatically retrieves and pre-fills the table ID from the <caption>
  • Dynamically toggles between English and French instantly updating markup upon selection
  • Creates a dedicated footnotes section below the table
